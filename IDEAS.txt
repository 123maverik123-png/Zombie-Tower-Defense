# ИДЕИ ДЛЯ РАЗВИТИЯ ИГРЫ (кроме баланса и графики)
# Составлено: 2026-07-20
# Отсортировано по соотношению "ценность / трудозатраты" внутри каждого раздела.
# Оценка: [S] — вечер, [M] — пара дней, [L] — неделя+

=====================================================================
1. ГЕЙМПЛЕЙНЫЕ МЕХАНИКИ
=====================================================================

[S] Ускорение времени (x2 / x3)
    Кнопка и хоткей. Поле state.game_speed уже существует в коде —
    осталось умножать dt в update и добавить кнопку в HUD.
    Самая частая просьба игроков в любой TD.

[S] Продажа и перестановка в межволновой паузе
    Продажа уже есть. Добавить: в паузу между волнами продажа
    возвращает 100% цены вместо обычных ~70% — поощряет
    эксперименты с расстановкой.

[M] Способности игрока (active skills)
    2-3 кнопки с кулдауном: метеорит по области, заморозка всех
    врагов на 3 сек, мгновенный ремонт ворот. Даёт игроку занятие
    ВО ВРЕМЯ волны (сейчас после расстановки башен остаётся только
    смотреть). EventBus и система эффектов уже готовы.

[M] Модификаторы волн
    Каждая 3-4 волна получает случайный модификатор, объявляемый
    заранее: "Быстрая волна" (+50% скорости), "Бронированная"
    (-30% физ. урона), "Ночная" (туман, радиус башен -1 клетка).
    Игрок видит модификатор и успевает перестроиться.

[M] Герой на карте
    Юнит, которого можно перемещать кликом по карте: медленно
    ходит, атакует ближайшего врага, качается между уровнями.
    Использует уже готовую систему анимаций врагов (4 направления).

[L] Ветки развития башен
    На 4-м уровне башня выбирает специализацию из двух:
    снайпер -> "Пробивной" (урон сквозь врагов) или "Охотник"
    (бонус к боссам). Удваивает глубину билдов без новых спрайтов —
    достаточно перекраски.

=====================================================================
2. МЕТАПРОГРЕССИЯ (между уровнями)
=====================================================================

[M] Дерево прокачки профиля
    За звёзды с уровней (1-3 за прохождение по числу оставшихся
    жизней) покупаются перманентные бонусы: +1 стартовая жизнь,
    +50 стартового золота, -5% цены апгрейдов. Profile и сейвы
    уже готовы, нужен только экран и таблица бонусов.

[M] Достижения
    "Пройди уровень без потери жизней", "Убей 1000 зомби огнём",
    "Пройди уровень только пулемётами". Дешёвая реализация через
    существующий EventBus (enemy_killed, tower_built уже
    эмитятся). Экран достижений в профиле.

[S] Статистика профиля
    Экран с цифрами: всего убито, любимая башня, потрачено золота,
    время в игре. Половина данных уже копится в player.stats —
    осталось сохранять в профиль и показывать.

=====================================================================
3. РЕЖИМЫ ИГРЫ
=====================================================================

[M] Бесконечный режим (Endless)
    Волны генерируются бесконечно с растущим HP-множителем
    (формула уже есть после ребаланса), таблица рекордов в
    профиле. WaveFactory почти не требует изменений — убрать
    ограничение num_waves и добавить счёт.

[S] Ежедневный вызов (Daily Challenge)
    Уровень с фиксированным random.seed(дата): у всех игроков
    одинаковая карта и волны. Генератор карт уже есть
    (level_generator.py).

[M] Испытания с ограничениями
    Готовые уровни с твистом: "только 5 башен", "без апгрейдов",
    "стартовое золото 100". Один JSON с ограничениями + проверка
    в towers_logic.build.

=====================================================================
4. ИНТЕРФЕЙС И УДОБСТВО (QoL)
=====================================================================

[S] Предпросмотр следующей волны
    Иконки состава волны в HUD ("8x обычный, 2x танк, летуны!").
    Данные уже лежат в wave_manager.wave_data — только отрисовать.
    Сильно влияет на осмысленность подготовки.

[S] Хоткеи покупки башен (1-9)
    Сейчас башня выбирается только мышью в панели.

[S] Кнопка "начать волну досрочно" с бонусом золота
    Пауза между волнами 3 сек фиксирована. Кнопка "GO" даёт
    +золото пропорционально сэкономленному времени. Стандарт
    жанра (Kingdom Rush).

[M] Мини-энциклопедия (бестиарий)
    Экран с описанием врагов и башен: характеристики, слабости,
    советы. Открывается по мере встречи врагов. Все данные уже
    в JSON-конфигах — интерфейс поверх них.

=====================================================================
5. КОНТЕНТ НА СУЩЕСТВУЮЩИХ СИСТЕМАХ
=====================================================================

[M] Новые типы врагов (без новых механик движка):
    - Лекарь: лечит зомби вокруг себя (аура)
    - Щитоносец: блокирует первые N попаданий
    - Разделяющийся: при смерти спавнит 2 маленьких
    - Сапёр: идёт к ближайшей башне и взрывается
    Первые два — только конфиг + немного кода в combat.

[M] Боссы с фазами
    На 25% HP босс ускоряется / призывает свиту / становится
    неуязвим на 3 сек. Сейчас босс — просто толстый зомби.

[S] Погодные пресеты для карт
    Дождь (вода дешевле), жара (огонь сильнее) — задаётся в
    levels_data.py per-level, влияет на цены/урон из конфига.

=====================================================================
6. ЗВУК И АТМОСФЕРА
=====================================================================

[S] Озвучка событий волны
    Горн при старте волны, колокол при прорыве к замку,
    фанфары при победе. Аудиосистема готова, нужны только файлы
    (generate_sounds.py уже умеет синтезировать звуки).

[M] Динамическая музыка
    Спокойный трек между волнами -> боевой во время волны.
    Кроссфейд через pygame.mixer.music + fadeout.

=====================================================================
РЕКОМЕНДУЕМЫЙ ПОРЯДОК (первая пятёрка):
=====================================================================
1. Ускорение времени [S]          — мгновенное улучшение ощущений
2. Предпросмотр волны [S]         — осмысленность решений
3. Кнопка ранней волны [S]        — темп + риск/награда
4. Бесконечный режим [M]          — реиграбельность за копейки
5. Способности игрока [M]         — активность во время волны

Логика порядка: сначала три дешёвых улучшения игрового цикла,
затем режим, который многократно окупает контент, затем механика,
которая занимает руки игрока в бою.
